##Personal notation for the structuring of tof_sensor.hpp##


Firstly has the #includes : 

#pragma once  //  this is standard practice to stop the file being included more than once in the same compilation.

#include <cstdint> // allows for fixed-width integer types i.e. uint8_t, uint16_t
#include <functional> // allows for std::function which is used for the callback
#include <thread>  // allows for std::thread, since this class runs a worker thread
#include <memory> // allows for smart pointers, i.e. shared_ptr
#include <gpiod.hpp> // this is for the libgpiod cpp interface, which is used to work with gpio lines in linux

#include "drivers/i2c_bus.hpp"  // so this class can use the I2CBus class for comms with the sensor


##

the class, of type ToFSensor is then defined - 

' class ToFSensor ' 

This class: 
- handles the VL53L1X sensor 
- uses a blocking GPIO interrupt to know when data is ready
- published the measured distance using a std::function callback
(This class is the main link between physical sensor and the rest of the system)


Unlike the test publisher (ToFPrinter) class, it is stateful i.e. has member variables, storing information over time.
 (which i2c bus its using, sensor address, what GPIO line, is it running, what callback has been registered, its worker thread)
Meaning its not just reacting to a value, it is managing the state and behaviour of the sensor system. 

##

Inside the public section
 ## 

Firstly :


' public: '

a type alias named 'DistanceCallbackInterface' for 'std::function<void(uint16_t)>; ' ,
which is a callable wrapper type that can store any function matching that signature.

This allows the system to store and use a callback function later, enabling the sensor to publish distance data without knowing how it will be used. 

It basically represents any callable object that takes a uint16_t (i.e. a distance in mm) and returns nothing. 

std::function = “a variable that holds behaviour instead of data” enables callbacks


'  using DistanceCallbackInterface = std::function<void(uint16_t)>;  '

' using ' is used to define the type alias (a nickname for the type), happens at compile time (no runtime cost)


Intuitively, ToFSensor producs data, it doesn't know what to do with it, so it asks 'give me a function to call when I have a result' ...
That function is stored using 'DistanceCallbackInterface'....


##

Then the constructor is declared: 

'ToFSensor(I2CBus& bus,
          uint8_t address = 0x29,
          const std::string& gpioChipPath = "/dev/gpiochip0",
          unsigned int gpioLine = 4); '

Which is called when a ToFSensor object is created. 

It takes: 

I2CBus& bus -> a reference to the I2C bus (not owned, just used)
uint8_t address -> the sensors I2C adress (0x29 in this case)
std::string gpioChipPath -> which GPIO chip to use (/dev/gpiochip0)
unsigned int gpioLine -> which GPIO line/pin is connected to the interrupt. 

This sets the object up with all the info it needs to interact with the hardware.
